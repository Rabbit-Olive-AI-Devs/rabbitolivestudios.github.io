# World Cup 2026 — Country Flags + Full Team Names (v3.14.0)

**Date:** 2026-06-24
**Status:** Approved design, pending implementation plan
**Scope:** Enhance the World Cup dashboard (`/worldcup` mono + `/color/worldcup` color) to (a) show full country names instead of 3-letter codes on both displays, and (b) add small country flags on the color display only.

---

## 1. Goals

1. Replace 3-letter team codes (`BRA`, `SUI`) with **full country names** (`Brazil`, `Switzerland`) everywhere on both displays. Truncate with an ellipsis per-layout when the name does not fit. Never fall back to the 3-letter code.
2. Add **country flags** next to team names on the **color display only** (E1002, Spectra-6). The mono display (E1001) stays text-only — flags are not legible in pure black/white.

## 2. Decisions & rationale

- **Flags on color only.** Spectra-6 (black/white/red/yellow/green/blue) is close to "flag colors", so flat flags map cleanly. On the mono display, red/blue/green fields all collapse to indistinguishable black — flags there are noise. Mono keeps full names only.
- **Flag source: lipis/flag-icons (MIT), pre-rendered offline (Option A).** flag-icons uses each flag's official hex colors and fine emblem detail — both fight a 6-color, ~24px, device-quantized display. So we use flag-icons as *source art* and pre-render each needed flag to a Spectra-6-dithered indexed PNG **at build time**, committing the result. This is the same philosophy the project's color image pipelines already use (server-side Floyd–Steinberg to the exact palette so the device renders what we send). The Workers runtime cannot rasterize SVG, so this step is offline only.
- **Truncated names, not codes, for the fallback.** When a full name does not fit a layout, truncate with `…` (e.g. `United State…`) rather than dropping to `BRA`. Per-layout character budgets, tuned against live data at 800×480.
- **No KV cache-key change.** Flags are static assets baked into the Worker bundle; the `wc:data:v1` data blob is unchanged. Rendering changes are pure presentation.

## 3. Flag asset pipeline (offline build)

New script `scripts/generate-flags.ts` (run on a dev machine / CI, **not** in the Worker):

1. **FIFA → ISO-2 map** for the 48 participating teams, as a static table in the script (`BRA→br`, `SUI→ch`, `USA→us`, `KOR→kr`, …). Scope is this tournament's participants only; the route is removed after 2026-07-19.
2. **Rasterize** each flag-icons `4x3` SVG to a small chip (~**36×27 px**) using `@resvg/resvg-js` (prebuilt binaries; lighter than `sharp`). devDependency only.
3. **Dither to Spectra-6** using the project's existing `src/dither-spectra6.ts` (Floyd–Steinberg) and encode with `encodePNGIndexed()` from `src/png.ts`. No new image code — reuse the proven path. (These modules are pure TS with no Workers-runtime APIs, so they import cleanly in Node.)
4. **Emit** a committed generated file `src/worldcup-flags.ts`:
   ```ts
   // GENERATED by scripts/generate-flags.ts — do not edit by hand.
   export const FLAGS: Record<string, string> = {
     BRA: "iVBORw0KGgo…", // base64 indexed PNG
     // …
   };
   ```

**Dependency impact:** `flag-icons` + `@resvg/resvg-js` are `devDependencies` only — zero runtime/bundle cost. The generated indexed PNGs are tiny (~200–400 B each × ~48 ≈ **~15 KB** of base64 in the bundle).

## 4. Runtime rendering changes (`src/worldcup-ui.ts`)

- **`teamLabel(team, maxChars)`** — new exported pure function. Returns the escaped full `team.name`, truncated to `maxChars` with a trailing `…`. Empty/placeholder names return `"TBD"`. Replaces `teamCode()` at all render call-sites (`matchRow`, `groupTable`, `bracketColumn`, `championLayout`). `teamCode()` remains only where a bare code is still needed internally (favorite-team comparison via `isFav`).
- **`WcTheme.flag?: (code: string) => string`** — new optional theme hook. Color theme returns `<img class="wc-flag" src="data:image/png;base64,…">` looked up from `FLAGS` (missing code → `""`). Mono theme omits it. This keeps `worldcup-ui.ts` display-agnostic — flags are just another theme-driven accent, like the existing `fav`/`win`/`live` colors.
- **`.wc-flag` CSS** (color page only): ~13 px tall, `width:auto`, hairline black border so white-heavy flags (Japan) separate from the white page background. Champion-card flag is a larger variant (~110 px tall).
- Favorite-team highlighting (`isFav`, `FAVORITE_TEAM = "BRA"`) is unchanged — it still compares 3-letter codes derived via `teamCode()`.

## 5. Layout-by-layout behavior

| Layout | Color display (E1002) | Mono display (E1001) |
|---|---|---|
| Today / Latest results rows | flag + full name (trunc ≈14) | full name (trunc ≈16) |
| Group table (team column) | flag + name (trunc ≈12) | name (trunc ≈14) |
| Bracket ties (4 tight columns) | small flag + name (trunc ≈9) | name (trunc ≈10) |
| Champion card | large flag (~110 px) above the big name | big name only |

Truncation budgets are starting points; final values tuned against live data at 800×480.

## 6. Edge cases

- **TBD / undecided knockout slots** (empty code/name): no flag, render `"TBD"`.
- **Missing FIFA→ISO entry or absent flag**: no flag; the name still renders. The page never breaks on a missing flag.
- **Long names** (`United States`, `South Korea`, `Saudi Arabia`): handled by per-layout truncation.
- **White-dominant flags** (Japan): hairline border preserves the chip boundary on white background.

## 7. Testing

- Unit tests (`tests/worldcup.test.js`):

- `npm run typecheck`, `npm run test:utils`, `npm run dry-run` all clean.
- Visual QA at 800×480 on the color panel (manual, device-side) — flag legibility and truncation fit, especially the dense bracket layout.
